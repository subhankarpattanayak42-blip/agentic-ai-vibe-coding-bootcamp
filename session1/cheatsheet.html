10 Essential Terms

🤖 Agent
An autonomous system that perceives its environment, makes decisions, and takes actions — without step-by-step human instruction.
Example: An AI that books a meeting by checking calendars, finding a slot, and sending invites.
🎛️ Orchestrator
The control layer that manages an agent's workflow — deciding which tools to use, in what order, with what context.
Example: The part of Claude Code that decides "first read the file, then edit, then run tests."
🔧 Tool
A function or API an agent can call to interact with the outside world — file system, database, web search, code execution.
Example: Bash tool lets an agent run terminal commands; Read tool reads files.
🧠 Context Window
The maximum amount of text (tokens) an LLM can "see" at once. Everything — instructions, history, tool results — must fit inside it.
Claude Sonnet has ~200K token context. 1 page ≈ 300–400 tokens.
🔄 Reasoning Loop
The iterative cycle agents run: Think → Plan → Execute Tool → Observe Result → Decide next step. Repeats until task is done.
Each loop costs tokens and API calls — efficiency matters.
💾 Prompt Caching
Technique to reuse expensive system prompts across calls — cached prompts cost ~10% of the original price.
Your CLAUDE.md instructions can be cached, saving 80%+ on long sessions.
📦 Compaction
When context gets full, older messages are summarised into a compact form — allowing the agent to continue without hitting the limit.
/compact command in Claude Code triggers this manually.
🏗️ Harness Engineering
The discipline of building the scaffolding around an LLM — tools, memory, orchestration, CLAUDE.md — to create reliable agentic systems.
Claude Code, Open Claw, and Hermes are all harness engineering tools.
🎵 Vibe Coding
Building functional software using natural language descriptions — you describe the outcome, the agent writes the code.
"Build me a PM dashboard with RAG status, filters, and CSV export" → working app.
📚 RAG
Retrieval-Augmented Generation — a pattern where an agent retrieves relevant documents before generating a response, grounding answers in real data.
Searching a knowledge base before answering a question.

Executive Communication: 3 Layers

Layer 1 — The Problem
What senior leaders actually care about
  • "We're paying highly skilled people to do work AI can automate"
  • "Our competitors using AI are shipping features in days, not weeks"
  • "Manual processes create compliance gaps and audit risk"
Layer 2 — The Solution
What agentic AI actually provides
  • Autonomous execution — AI completes multi-step tasks without hand-holding
  • Self-correcting reasoning — fewer escalations, fewer bugs, fewer re-works
  • End-to-end workflow automation, not just individual task automation
Layer 3 — Business Impact
The numbers that justify investment
  • Capacity recapture: "If agents do 30% of your team's work, that's 30% more capacity at zero extra headcount"
  • Speed: "3-week feature delivery → 3 days with agentic tooling"
  • Quality: "Automated checks mean consistent, auditable outputs every time"

Power Phrases — Use These in Any Meeting

Agentic AI doesn't replace people — it removes the drudgery so people focus on judgment.
The question isn't whether to adopt agentic AI — it's whether we adopt it before our competitors do.
We're not talking about automation. We're talking about a force multiplier for every professional.
These tools don't need perfect prompts — they need clear goals. That's a human skill.
