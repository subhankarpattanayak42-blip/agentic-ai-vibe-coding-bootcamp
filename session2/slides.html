What is Open Claw03 / 10

Open Claw — Positioning

Open Claw is an open-source agentic CLI that runs autonomous coding tasks in your terminal — reading files, writing code, running commands, browsing the web — all directed by natural language.

vs Claude Code (Anthropic)

  • Claude Code: closed, polished, opinionated
  • Open Claw: open-source, transparent, forkable
  • Same concept — different philosophy
  • Open Claw = best for learning internals

Key Capabilities

  • Read & write any file in your project
  • Execute shell commands safely
  • Browse web + search documentation
  • Multi-step reasoning with tool calling
  • Bring-your-own model / API key
💡 Key Mental Model

Open Claw is not a chatbot. It's an agent — it takes a goal and figures out the steps. You give it outcomes, not instructions.

Token Flow05 / 10

What Fills Your Context Window

Context is consumed by:

  • System prompt / AGENTS.md (big!)
  • Conversation history (grows every turn)
  • Tool results (file reads, bash output)
  • Model's reasoning + responses

Tool Call Example:

User: "Find all TODO comments"
→ Agent: run Bash: find . -name "*.py"
← Result: 47 file paths [+tokens]
→ Agent: read each file...
← Result: each file content [+++tokens]
→ Context filling fast!

Context Breakdown (typical 100K task)

System (25%)
Tool Results (30%)
History (25%)
Response (20%)

Token Efficiency06 / 10

3 Efficiency Mechanisms

MechanismHow It WorksWithoutWith
Prompt Caching System prompt cached after first call; subsequent calls use cache at ~10% cost 10 × full cost ~1.9× total cost
Compaction At ~80% context capacity, old turns summarised into compact form. Agent continues without reset. Task fails at limit Unlimited task length
Targeted Reads Grep first, then read only matching lines/files. Never read entire codebase into context. All 200 files = 400K tokens 3 relevant files = 4K tokens
💰 Real Savings

A typical day of 50 agentic tasks with a 10K-token system prompt: without caching = ₹500, with caching = ₹95. Same results, 80% cheaper.

Do's & Don'ts07 / 10

Rules for Safe, Effective Use

✅ DO

  • Write a clear, specific AGENTS.md
  • Give specific tasks: "fix auth.py:45"
  • Enable prompt caching for long sessions
  • Use /compact before context fills
  • Review each tool call in sensitive environments
  • Store secrets in environment variables
  • Commit your code frequently
  • Use skills for repetitive multi-step tasks
  • Test your setup with a simple task first
  • Set a token budget per task

❌ DON'T

  • Give vague tasks like "improve the code"
  • Let agent run unattended on production
  • Read entire codebase into context at once
  • Ignore tool call permission prompts
  • Put API keys in AGENTS.md or prompts
  • Trust first output without reviewing
  • Run destructive bash commands unsandboxed
  • Forget to set a context/cost limit
  • Mix instructions and data in one prompt
  • Use agentic tools for human-judgment tasks

Runtime Architecture09 / 10

Node.js & Python — The Harness Split

Open Claw and Claude Code both use two runtimes working together. Node.js owns the shell; Python owns the automation scripts.

🟢 Node.js — The Harness Shell
CLI & REPL Loop
The harness boots in Node.js — it manages your terminal session, keyboard input, and the agent's plan→act→observe loop.
Built-in Tool Execution
Read, Write, Edit, Bash, WebFetch are all Node.js modules. When Claude calls a tool, Node.js runs it and feeds the result back.
MCP Server Spawning
Model Context Protocol servers are child processes launched by Node.js — connecting the agent to databases, APIs, and external services.
Hooks & Permissions
Pre/post-tool hooks in settings.json are shell commands fired by Node.js's child_process module. Permission gates live here too.
🐍 Python — The Automation Layer
Custom Skill Scripts
Every /skill in this bootcamp is a Python script — proposal generation, RFP analysis, Wrike automation. Node.js spawns them as subprocesses.
Document & Output Generation
python-docx, openpyxl, python-pptx generate Word docs, Excel sheets, and PowerPoint decks. The output layer is entirely Python.
Anthropic SDK Calls
Scripts call anthropic.Anthropic() directly to invoke Claude models — a separate API session from the harness's own model calls.
Browser Automation (Playwright)
Playwright-Python drives Chrome via CDP for skills that scrape content libraries or post comments to project management tools.
🔄 How They Work Together

You type /skill → Node.js harness routes it → spawns Python subprocess → Python calls Anthropic API + generates output → Node.js surfaces result back to your session.
