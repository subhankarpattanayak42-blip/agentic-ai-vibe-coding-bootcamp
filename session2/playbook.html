Architecture at a Glance

User Input (natural language) ↓ Harness Layer ── CLAUDE.md (your instructions) ── settings.json ── hooks ↓ Orchestrator ── Main reasoning loop: Plan → Act → Observe → Evaluate ↓ Tool Registry: [Read] [Write] [Edit] [Bash] [WebFetch] [WebSearch] ↓ LLM (Claude Sonnet / Opus) ── Anthropic API ── your API key ↓ Response + tool_use call → harness executes → tool_result returned to LLM ↑________________________________ loop until task complete ______________|

Key insight: Every tool result adds tokens to context. Large bash outputs or full file reads fill the window fast. Use targeted reads.

Setup Guide

# 1. Install Node.js (v18+) from nodejs.org

# 2. Install Claude Code (Open Claw equivalent)
npm install -g @anthropic-ai/claude-code

# 3. Set your API key (get from console.anthropic.com)
export ANTHROPIC_API_KEY="sk-ant-your-key-here"
# Add to ~/.zshrc or ~/.bash_profile to persist

# 4. Verify installation
claude --version

# 5. Navigate to any project folder and launch
cd ~/my-project
claude

# 6. (Optional) Create a CLAUDE.md in your project root

Token Efficiency — Your Settings

MechanismHow to EnableBenefit
Prompt CachingAutomatic if using Claude models — ensure your CLAUDE.md is stable (don't change it every session)~80% cost reduction on repeated tasks
CompactionType /compact when context is getting full, or enable auto-compact in settingsUnlimited task length without context errors
Targeted ReadsAsk for specific line ranges: "read lines 40-80 of auth.py" — never "read all files"90% fewer tokens on large codebases

Do's & Don'ts — Quick Reference

✅ DO

  • ✓ Write a clear, specific CLAUDE.md
  • ✓ Give specific tasks with file paths
  • ✓ Enable prompt caching
  • ✓ Use /compact proactively
  • ✓ Review tool calls before executing
  • ✓ Store secrets in env variables
  • ✓ Commit code frequently
  • ✓ Use skills for repetitive tasks
  • ✓ Test setup with a simple task first
  • ✓ Set a token/cost budget

❌ DON'T

  • ✗ Give vague tasks ("improve the code")
  • ✗ Run unattended on production
  • ✗ Read entire codebase at once
  • ✗ Ignore permission prompts
  • ✗ Put API keys in prompts
  • ✗ Trust first output blindly
  • ✗ Run destructive commands unsandboxed
  • ✗ Forget context limits
  • ✗ Mix instructions and data
  • ✗ Use for human-judgment tasks
