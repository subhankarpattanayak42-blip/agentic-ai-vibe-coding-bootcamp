What is Open Claw?

Open Claw is an open-source agentic CLI tool that provides an autonomous coding assistant running directly in your terminal. It connects to AI models (primarily Claude) and executes multi-step development tasks by calling tools, reading files, writing code, and running commands.

Cron Jobs — Scheduling Agent Tasks

Claude Code supports scheduled tasks natively via its built-in CronCreate tool — no external tooling needed. You describe a recurring task in natural language and set a cron expression; the agent fires on schedule.

Use CaseCron ExpressionWhat it does
Daily standup report0 9 * * 1-5Runs every weekday at 9 AM
Weekly project summary0 8 * * 1Runs every Monday at 8 AM
Status check every 30 min*/30 * * * *Polls a task or dashboard every 30 min
End-of-day wrap-up0 17 * * 1-5Runs weekdays at 5 PM

Open Claw uses system cron (crontab -e) to schedule open-claw --prompt "your task". Codex has no scheduling support. Claude Code's native cron is the most powerful option for PM automation workflows.

Under the Hood — Token Flow & Tool Calling

The Context Window

Every LLM has a finite context window (token limit). Open Claw manages this by deciding what to include: system prompt, conversation history, tool results, and file contents. As context grows, older content gets compressed or discarded.

Tool Calling Flow

1. User gives task: "Read all .py files and find TODOs"
2. Model reasons: "I need to list files first"
3. Model emits: tool_use { name: "Bash", input: { command: "find . -name '*.py'" } }
4. Harness executes the command
5. Result returned to model as tool_result
6. Model reasons over result: "Now read each file..."
7. Model emits next tool_use
8. Loop continues until task complete or context exceeded

Key Insight: The Model Sees Everything

Every tool result is added to the conversation. If you run many large file reads, the context fills up rapidly — this is why token efficiency matters so much in agentic systems.

4:25–4:35
Token Efficiency Mechanisms

1. Prompt Caching

Anthropic's API supports cache breakpoints. If your system prompt (e.g., AGENTS.md instructions) doesn't change, it can be cached — serving it costs ~10% of the original token price. For a 50-session workday, this can cut costs by 80%+.

2. Context Compaction

When the context window fills up, Open Claw can summarise older turns and replace them with a compact summary. The agent continues working without losing the thread, but older verbatim content is discarded.

# Compaction trigger in Open Claw:
# When context reaches ~80% capacity:
# Older messages → summarised → summary replaces raw messages
# Recent messages + summary = new effective context

3. Strategic File Reading

Rather than reading entire codebases, efficient agents use targeted reads (line ranges, grep results) and only pull full files when necessary. Every unnecessary token is wasted money.

Token Cost Comparison

ScenarioWithout OptimisationWith Caching + Compaction
10 tasks, same system prompt10 × full prompt cost1 × full + 9 × 10% = ~1.9× cost
Long multi-step taskContext fills, failsCompaction extends capacity
Large codebase readReads all files = expensiveGrep + targeted reads = 90% less

Do's & Don'ts

✅ DO

  • Write a clear, specific AGENTS.md with context about your project
  • Use targeted prompts — "fix the login bug in auth.py:45" not "fix bugs"
  • Enable prompt caching for long sessions
  • Use /compact proactively before hitting context limits
  • Review each tool call before it executes in sensitive environments
  • Use --dangerously-skip-permissions only in sandboxes
  • Store secrets in environment variables, never in prompts
  • Test your AGENTS.md with a simple task first
  • Use skills (slash commands) for repetitive multi-step tasks
  • Commit frequently — the agent may make unexpected changes

❌ DON'T

  • Give vague tasks like "improve the code" — you'll get random refactoring
  • Let the agent run unattended on production systems
  • Feed the entire codebase into context at once
  • Ignore tool call permissions — they protect you
  • Store API keys in AGENTS.md or prompt text
  • Trust the agent's first output without reviewing it
  • Use agentic tools for tasks requiring human judgement (legal, medical)
  • Run destructive bash commands without sandbox
  • Forget to set a context limit — runaway agents cost money
  • Mix instructions and data in the same prompt block

Bonus: Skills
Skill Building in Open Claw — Claw Hub & Custom Commands

What Are Skills?

Skills (also called slash commands) are reusable, named workflows you invoke with a /command shortcut. Instead of typing a long prompt every time, you define it once and call it in one word. Open Claw loads skills from a .open-claw/commands/ folder in your project.

Claw Hub — The Skill Marketplace

Claw Hub is Open Claw's community skill library — browse, install, and share skills built by other users. Think of it like an app store for agent workflows.

How Claw Hub works:

Browse skills at hub.openclaw.ai
    ↓
Install a skill: openclaw skills install <skill-name>
    ↓
Skill saved to .open-claw/commands/<skill-name>.md
    ↓
Use it instantly: type /skill-name in any session
    ↓
Agent reads the skill file and executes the workflow

Anatomy of a Skill File

# skill: daily-standup
# description: Generate a standup update from recent git commits and open tasks

Look at the last 5 git commits using Bash.
Check for any open TODO comments in the codebase.
Then write a concise standup update in this format:
- What I did yesterday
- What I'm doing today
- Any blockers

Keep it under 100 words. Use bullet points.

5 Ready-to-Use Skill Examples

/daily-standup

Reads git log + open tasks → formats a 3-bullet standup in seconds. Great for Monday morning.

/daily-standup

/code-review

Reviews changed files vs main branch — flags bugs, style issues, and missing tests. Instant Pink Team.

/code-review

/write-readme

Reads the codebase, understands the project, and generates a professional README.md with setup instructions.

/write-readme

/bug-hunt

Scans all source files for error-prone patterns, uncaught exceptions, and missing validations. Returns a ranked bug list.

/bug-hunt

/pm-status-report [your most powerful skill]

Reads your project folder, extracts task statuses, calculates % complete, flags overdue items, and emails a formatted status report. One command replaces 45 minutes of manual work.

/pm-status-report --project "Q3 Rollout" --send-to manager@company.com

Pro Tips for Skill Design

  • One skill = one job — keep each skill focused on a single, repeatable workflow
  • Use arguments — skills can accept $ARGUMENTS to make them flexible (e.g. /summarise --file report.pdf)
  • Chain skills — run /research first, then /write-email to build multi-step pipelines
  • Version control your skills — commit .open-claw/commands/ so the whole team shares the same skill library
  • Share on Claw Hub — once a skill is useful, publish it for the community with openclaw skills publish
