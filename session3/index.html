What is Vibe Coding?

Vibe coding is the practice of building fully functional software using natural language instructions — describing what you want and letting an agentic AI build it. No syntax memorisation. No boilerplate. Just describe your vision and the agent writes, runs, and refines the code.

The Shift

Traditional CodingVibe Coding
Write every line manuallyDescribe the outcome, agent writes
Requires syntax expertiseRequires clarity of thought
Debug line by lineAgent self-corrects with your feedback
Weeks to a working prototypeHours or less to a working app
Barriers: learning curveBarrier: knowing what you want

PM App Ideas — Pick Your Build

Each participant picks one app to build. All can be completed in ~20 minutes with vibe coding.

Live Build: PM Status Dashboard

Phase 1 (4:20–4:38): The instructor builds a PM Status Dashboard live using 3 prompts — participants watch, take notes, and copy prompts. Phase 2 (4:38–4:45): Participants apply the same technique to their chosen app (7 minutes to get a first version running). The goal is a working first draft, not a finished product.

Step 1 — Initial Prompt (4:20)

Prompt 1
"Create a single-file HTML project status dashboard. It should have a header with the title 'Project Status Dashboard', a table showing: Project Name, Status (with RAG colour: Red/Amber/Green), Owner, Due Date, and % Complete. Add 5 sample projects. Make it look professional with a dark navy background and gold accents. Include a filter dropdown to filter by status."

Step 2 — Refinement (4:30)

Prompt 2
"Now add: (1) A summary bar at the top showing total projects, how many are Green/Amber/Red. (2) An 'Add Project' button that opens a form to add new rows. (3) Click any row to edit it inline. (4) A 'Export CSV' button."

Step 3 — Polish (4:38)

Prompt 3
"Make it mobile-responsive, add a footer with today's date and 'Built with AI', and add smooth animations when rows are added or status changes."

Participant Guidance During Build

  • Adapt each prompt for your chosen app type
  • Use follow-up prompts to fix anything that doesn't look right
  • Don't worry about understanding every line of code — focus on outcomes
  • Save your HTML file to Desktop as my_pm_app.html
Starter Prompts Library
Ready-to-Use Vibe Coding Prompts

Use these as starting points for any PM app:

Prompt A — Generic PM App Starter
"Build a single-file HTML [APP TYPE] tool. Include: [FEATURE 1], [FEATURE 2], [FEATURE 3]. Style: dark navy background (#0D1B2A), gold accents (#F0C040), clean modern typography. Make it fully functional with no backend required — use localStorage to persist data."
Prompt B — Add Persistence
"Update the app to save all data to localStorage so it persists when the page is refreshed. Show a 'Saved ✓' indicator when data is auto-saved."
Prompt C — Add Export
"Add a button to export all data as a CSV file. Name the download file '[APPNAME]_export_[today's date].csv'."
Prompt D — Polish and Responsive
"Make the layout fully responsive for mobile. Add smooth CSS transitions. Add a professional header with the app name and today's date. Add a footer crediting 'Built with AI by [Your Name]'."
Prompt E — Debug and Fix
"The [FEATURE] is not working correctly — when I [ACTION], it should [EXPECTED RESULT] but instead [ACTUAL RESULT]. Please fix this."
