How to Use These Prompts

Each prompt is a template. Replace [BRACKETED WORDS] with your specific details. Start with Prompt A for any new app, then layer in B, C, D to enhance it. Use Prompt E whenever something doesn't work as expected.

Prompt A
Generic PM App Starter
Use this FIRST for any new app
"Build a single-file HTML [APP TYPE] tool. Include: - [MAIN TABLE/VIEW] with columns: [COL1, COL2, COL3, COL4] - [KEY FEATURE 1] - [KEY FEATURE 2] - Add [NUMBER] sample rows of realistic data Style: dark navy background (#0D1B2A), gold accents (#F0C040), clean modern typography. Make it fully functional with no backend — use localStorage to persist data."
  • Name every column you want — vague prompts produce vague apps
  • Ask for sample data — it makes the app immediately usable and easier to review
  • Specify colours — otherwise you get default browser styling
Prompt B
Add Persistence + Summary
Use after Prompt A — adds data saving and summary stats
"Update the app to: 1. Save all data to localStorage so it persists when the page is refreshed. Show a 'Saved ✓' indicator that appears briefly after any change. 2. Add a summary bar at the top showing: total items, count by [STATUS FIELD] (e.g., how many Red/Amber/Green). 3. Add an 'Add New [ITEM]' button that opens a clean form — save on submit, cancel on close."
  • Always add localStorage in step 2 — losing data on refresh kills adoption
  • Summary stats make dashboards feel real and useful immediately
Prompt C
Add Export + Filtering
Use to make the app shareable and navigable
"Add to the existing app: 1. An 'Export CSV' button that downloads all data as a CSV file named '[APPNAME]_export_[today's date].csv'. 2. A filter/search bar at the top that filters rows in real time as you type. 3. A dropdown filter for [STATUS/CATEGORY FIELD] showing all unique values. 4. A 'Clear Filters' button to reset to full view."
  • Export makes the app useful beyond a browser — share in email or open in Excel
  • Real-time filter is the #1 requested feature in PM tools
Prompt D
Polish, Responsive + Footer
Use as the final polish pass before sharing
"Final polish pass on the app: 1. Make the layout fully responsive for mobile screens (cards stack on small screens, table scrolls horizontally). 2. Add smooth CSS transitions: rows slide in when added, status badges animate on change. 3. Add a professional header with the app name and today's date auto-updating. 4. Add a footer: 'Built with AI by [Your Name] · [Date]'. 5. Ensure the colour scheme is consistent throughout — use CSS variables."
  • CSS variables make future colour changes a one-line edit
  • Mobile responsiveness is a must — participants often check apps on phones
Prompt E
Debug & Fix Formula
Use whenever something doesn't work as expected
"The [FEATURE] is not working correctly. When I [DESCRIBE THE ACTION YOU TAKE], it should [DESCRIBE THE EXPECTED RESULT] but instead [DESCRIBE WHAT ACTUALLY HAPPENS]. Please fix this. Do not change any other features."
  • The last sentence ("do not change other features") prevents unwanted side effects
  • Be specific about the action — vague bug reports produce vague fixes
  • If it still doesn't work, paste the exact error message or describe the visual glitch

6 PM App Specs — Full Column Lists

📊
Status Dashboard
Most Popular
Columns: Project Name, Status (RAG), Owner, Due Date, % Complete, Notes
  • RAG colour coding (Red/Amber/Green)
  • Filter by status dropdown
  • Summary bar: total + RAG counts
  • Inline edit on row click
  • Export to CSV
✅
Task Register
Columns: Task, Owner, Priority (High/Med/Low), Due Date, Status, Notes
  • Overdue rows highlighted red
  • Priority badge colours
  • Filter by owner and priority
  • Mark complete button per row
  • Completed items strike-through
⚠️
Risk Register
Columns: Risk Description, Category, Probability (1-5), Impact (1-5), Risk Score, Owner, Mitigation
  • Auto-calculate: Score = P × I
  • Colour by score: >15 Red, 8-15 Amber, <8 Green
  • Sort by risk score descending
  • Category filter dropdown
  • Risk matrix 5×5 visual
📅
Agenda Builder
Fields: Meeting Title, Date, Attendees, Agenda Items (topic, owner, time, type)
  • Time slot auto-calculation
  • Drag to reorder agenda items
  • Running total time display
  • Print-optimised layout
  • Copy as plain text button
💰
Budget Tracker
Columns: Workstream, Budget (₹), Spent (₹), Committed (₹), Variance, Status
  • Auto-calculate variance
  • Chart: budget vs spent bar chart
  • Over-budget rows highlighted red
  • Total row at bottom
  • Currency formatting (₹ or $)
🗺️
Roadmap View
Fields: Milestone, Owner, Start Date, End Date, Status, Track/Workstream
  • Gantt-style horizontal bars
  • Colour by status (RAG)
  • Today line marker
  • Group by workstream/track
  • Milestone dependency arrows
