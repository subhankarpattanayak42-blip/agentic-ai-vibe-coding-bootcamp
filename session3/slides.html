Live Build — Step 105 / 10

Build Prompt #1 — Foundation

Your opening prompt sets up the entire app. Be specific about layout, features, and style.

Prompt 1 — PM Status Dashboard (adapt for your chosen app)
"Create a single-file HTML project status dashboard. Include: a header with the title 'Project Status Dashboard', a table showing Project Name, Status (RAG colour: Red/Amber/Green), Owner, Due Date, and % Complete. Add 5 sample projects. Style: dark navy background (#0D1B2A), gold accents (#F0C040), clean modern typography. Add a filter dropdown to filter by status."
💡 Prompting Tips

Name specific colours · List every column you want · Give sample data · State the style. The more specific, the closer you get on the first pass.

Build Prompt #2 — Features

Once the foundation is working, layer in interactive features.

Prompt 2 — Add Interactivity
"Add: (1) A summary bar at the top showing total projects + count of Green/Amber/Red. (2) An 'Add Project' button opening a form to add rows. (3) Click any row to edit it inline. (4) An 'Export CSV' button that downloads all data."
Prompt 3 — Polish
"Make it mobile-responsive. Add smooth CSS animations when rows are added or status changes. Add a footer with today's date and 'Built with AI by [Your Name]'. Save all data to localStorage so it persists on refresh."
🔄 The Iteration Loop

Describe → Review → Refine. If something doesn't look right, describe the issue in plain English. The agent will fix it.

Handling Issues07 / 10

When Things Don't Look Right

The Debug Prompt Formula

"The [feature] is not working correctly. When I [action], it should [expected result] but instead [actual result]. Please fix this."

Common Issues + Fixes

  • Filter not working → describe exactly what value and what you expect
  • Colours wrong → paste the hex codes you want
  • Layout broken on mobile → "make this mobile responsive"
  • Data not saving → "add localStorage persistence"

Refinement Philosophy

Vibe coding is iterative. Your first prompt gets you 70% there. Subsequent prompts refine. Expect 3–5 rounds for a polished result.

Key Mindset

You are the product manager, the agent is the developer. Your job is to describe outcomes clearly — not to write code.

GitHub Pages — Quickest Steps

1. Create repo at github.com (public, enable Pages in Settings)
2. Upload your .html file as index.html
3. Go to Settings → Pages → Source: main branch → Save
4. Your URL: https://[username].github.io/[repo-name]/
